## Описание реализации

### TaskQueue (queue.py)

Класс пользовательской коллекции задач.

Поддерживает:
- итерацию (`for task in queue`);
- получение длины (`len(queue)`);
- проверку наличия (`task in queue`, `id in queue`);
- добавление задач (`add_task`, `extend`).

Очередь хранит только объекты `Task`, выполняется валидация входных данных.

---

### Фильтры (filters.py)

Реализованы функции ленивой фильтрации:

- `filter_by_status`
- `filter_by_priority`
- `filter_ready_tasks`
- `filter_completed_tasks`

Используется `yield`, благодаря чему:
- не создаются лишние списки;
- обработка происходит по мере итерации;
- можно работать с любыми итерируемыми объектами.

---

### Исключения (queue_exceptions.py)

Реализованы пользовательские ошибки:

- `TaskQueueError` — базовый класс;
- `InvalidTaskError` — некорректный объект;
- `EmptyQueueError` — операции с пустой очередью.

---

### Интеграция

Решение интегрировано с предыдущими лабораторными:

- задачи приходят из источников (`GeneratorSource`, `FileSource`, `ApiSource`);
- используется функция `collect_tasks`;
- очередь оборачивает полученные данные и даёт удобный API.

---

### main.py

Реализована демонстрация:

- получение задач из генератора;
- создание очереди;
- вывод задач;
- применение фильтров.
